⚙️ How the Model Works

1. Preprocessing: Your leaf image is resized to 224×224 and normalized using ImageNet mean/std (required for the pre-trained backbone).

2. Feature Extraction: A frozen MobileNetV2 backbone (trained on 1.2M ImageNet images) extracts 1280 visual features via convolutional layers.

3. Classification: A fine-tuned dense head maps features → 38-class softmax, giving P(disease | image) for each class.

4. Uncertainty Check: If confidence <60% or Shannon entropy >2.5, the model flags the result as uncertain and recommends expert consultation.

📉 Bias-Variance Diagnostic — Loss Curves
Train loss vs validation loss. A widening gap (val >> train) signals overfitting (high variance). Converging curves indicate a well-regularized model. Dropout + early stopping prevent divergence.
📈 Accuracy Curves — Supervised Learning Progress
Both curves should rise and converge. If training accuracy is high but val accuracy plateaus, the model is memorizing training labels (overfitting). Early stopping triggers here.
⚡ Learning Rate Schedule
ReduceLROnPlateau halves the learning rate when val loss stops improving (patience=3). This is adaptive hyperparameter tuning — an informed search over the learning rate schedule.
🔵 Probability Distribution — P(disease | image)
The softmax output from the last prediction. A peaked distribution = high confidence. A flat distribution = high entropy → uncertainty flag → recommend expert consultation.
🏗️ Model Architecture — Transfer Learning

Backbone: MobileNetV2 (pre-trained, ImageNet)

Input: 224 × 224 × 3 RGB tensor (normalized)

Feature map: 1280-dimensional embedding (global avg pool)

Head: BatchNorm → Dropout(0.4) → FC(512) → FC(38)

Frozen layers: MobileNetV2 blocks 0–15 (universal features)

Fine-tuned: Last 3 blocks + custom head (disease features)

Optimizer: SGD(lr=0.01, momentum=0.9, wd=1e-4)

Loss: Cross-entropy with class weighting

📚 Course Concepts Applied in KrishiDrishti

Unit 1
Intelligent Agent
The app IS an agent. PEAS framework: Performance=F1 accuracy; Environment=farm field photos; Actuators=Hindi diagnosis output; Sensors=phone camera. Rationality: maximize correct diagnoses given observed images.
→ classifier.py: InferenceEngine.predict()
Unit 2
Search (Hyperparameter Tuning)
Finding optimal dropout rate, learning rate, and fine-tune blocks is a search problem over hyperparameter space. Grid search = uninformed; Bayesian optimization = informed search with a surrogate model. ReduceLROnPlateau is an adaptive informed search.
→ app.py: Trainer.fit() + LR scheduler
Unit 3
Probability Theory
Softmax outputs form a proper conditional probability distribution P(disease_k|image). Shannon entropy H(P) = -Σ p·log(p) measures prediction uncertainty. Low entropy → confident; high entropy → uncertain → flag for expert review.
→ classifier.py: predict_proba(), entropy calc
Unit 3
Linear Algebra / Optimization
The dense classification head is a linear transformation: logits = W·x + b. Training minimizes L(W,b) = -log P(correct class) via stochastic gradient descent. Gradient clipping prevents exploding gradients in deep networks.
→ classifier.py: CropDiseaseClassifier.forward()
Unit 4
Supervised Learning
Classic supervised classification: labelled pairs (leaf_image, disease_label). The model learns mapping f: X→Y that generalizes to unseen images. 80/10/10 train/val/test split. The test set is touched only once — the unbiased generalization estimator.
→ classifier.py: build_dataloaders(), Trainer
Unit 4
Overfitting & Regularization
Four anti-overfitting mechanisms: (1) Data augmentation expands effective dataset size. (2) Dropout randomly zeroes neurons during training (Dropout=0.4). (3) L2 weight decay penalizes large weights. (4) Early stopping halts training when val loss diverges.
→ classifier.py: get_transforms(), Trainer.fit()
Unit 4
Bias-Variance Tradeoff
Diagnostic: train_loss>>val_loss → underfitting (high bias); val_loss>>train_loss → overfitting (high variance). MobileNetV2 hits the sweet spot for our dataset size. Dropout rate and fine_tune_blocks are the primary knobs on this tradeoff.
→ Training tab: Loss Curves chart
Unit 4
Bayesian Statistics
Model outputs posterior P(disease|image) ∝ P(image|disease)·P(disease). Class weights encode the prior P(disease) from seasonal prevalence data. Uncertain predictions (low posterior, high entropy) are flagged — Bayesian caution applied to high-stakes agricultural advice.
→ classifier.py: InferenceEngine.predict()
Unit 4
Feature Learning
CNNs learn hierarchical representations: edge detectors (layer 1) → texture patterns (layer 5) → disease-specific lesion shapes (layer 16+). Global average pooling compresses the 7×7×1280 feature map to a 1280-d vector — the model's learned representation of the leaf.
→ classifier.py: CropDiseaseClassifier.features
Case Study
Transfer Learning
Pre-trained MobileNetV2 backbone (ImageNet) is frozen. Only the last 3 blocks + custom head are fine-tuned on crop disease images. This transfers universal visual knowledge to the agricultural domain — requiring only ~5,000 labelled examples instead of millions.
→ classifier.py: CropDiseaseClassifier.__init__()
Unit 4
Curse of Dimensionality
A raw 224×224×3 image has 150,528 dimensions. CNNs solve this by projecting images onto a low-dimensional disease-relevant manifold via convolutional weight sharing. Transfer learning further compresses this — the backbone already maps images into a compact 1280-d representation.
→ classifier.py: self.pool, self.features
Unit 4
Estimators & Macro F1
Accuracy alone is biased when class frequencies differ (rust is 10× more common than mosaic virus). Macro F1 = mean F1 across all classes = (2·Precision·Recall)/(P+R). This is an unbiased performance estimator that gives equal weight to rare diseases — critical for real-world impact.
→ classifier.py: evaluate_model()
